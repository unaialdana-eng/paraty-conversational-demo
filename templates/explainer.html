01

Why this exists

Purpose

Move 1 of the product agenda, running — not on slides.

The CPO deck proposes a conversational AI layer over Paraty's direct-booking engine as the first move of the 12-month plan. This demo is that move, end-to-end, so the conversation can happen on an artefact rather than on abstractions.

What it proves

The two-pass LLM pattern is shippable and robust.

Claude parses natural-language travel intent into structured JSON. Python deterministically filters and ranks inventory. Claude re-enters to write hotelier-friendly justifications and run the concierge sidebar. Structured where it matters, language where it matters.

Who it's for

Founders, product, and engineering — same artefact, different lenses.

Founders see direct-booking value and tone. Product sees the conversational surface and upsell hooks. Engineering sees the agent orchestration pattern, cached LLM outputs, and graceful fallback when the model is unavailable.

03

What's mock, what isn't

No real Paraty Tech API — 60 hand-curated mock hotels in hotels.json
The inventory mirrors Paraty's real geo footprint (Spain, Portugal, Italy, Morocco, Greece, LatAm, Japan). Hotel Kent and The Colony Hotel — real case-study properties from the public Paraty site — are anchors; the remaining 58 are plausible but fictional. In production this layer would hit the Paraty BE API.
Booking.com comparison is illustrative, not a live feed
Direct prices are stored per hotel. OTA prices use +12–20% deltas per industry benchmarks — realistic but not live. The savings headline is the right order of magnitude; the exact euros are for visual intent.
No authentication, no payment, no persistence
Checkout renders Apple Pay and guest fields; clicking "Complete reservation" does nothing. Out of scope: login, Stripe integration, session persistence, PMS write-back. All of these are trivial to add against a real backend — they're just not this prototype's point.
Claude drives the language, templates drive the rest of the concierge
On the Results page, the opening concierge bubble is Claude-generated per query; the upsells and cross-sells are parameterised templates keyed off the parsed intent. On the Property page, Claude generates the owner voice, walkable landmarks, proactive questions, contextual alert, and add-ons in a single call per hotel. If the Anthropic key is absent, every section has a deterministic fallback so the flow never breaks.
Out of scope for this prototype
Multi-agent handoff with clarifying questions (Move 2), WhatsApp / messaging channels (Move 3), MCP server exposure (so Perplexity or ChatGPT can query Paraty inventory directly), real PMS / CRS integration. These are the next 90–120 days of a 4-engineer + 1 PM squad; they're drawn in the deck, not demonstrated here.
Legal / IP
No Paraty proprietary information used. Non-compete (HomeToGo 2024–present): this demo does not use any HomeToGo IP or architecture. Non-compete (HBX 2023): more than 16 months elapsed, Paraty Tech is not an HBX-covered entity. If at any point Paraty asks to take this down after the conversation, it comes down same-day.
